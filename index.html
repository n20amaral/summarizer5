Version Control Systems (VCS)

What is it?

It's a system that tracks file changes and manages collaborative work on a project.

Why should we use it?

With a VCS it's possible to rollback to a previous version of a file and it's easier to solve conflicts when 2 or more people are working in the same document.
Brevly explain what isa VCS

No Version Control

What's wrong with not using version control?

Keep every version of a file will consume much more storage space. In a shared envinroment the file will be locked when someone is using it, making the others wait to finish. Merging local copies of a file manually will be a pain in the ass...
Simulate a no version control scenario

Centralized VCS

  • Require a connection to the central repository
  • Needs to solve merge conflicts before commit a file.
  • Offline and remote development it's hard to manage
Examples of CVCS: Subversion, CVS, Sourcesafe
Talk about personal experience

Distributed VCS

  • Does not require a central repository
  • Commits are made to local repository and all commits can be pushed to a remote repository
  • Can be connected to multiple remote repositories.
  • Offline and remote development friendly
Examples of DVCS: Git, TFS, Mercurial
